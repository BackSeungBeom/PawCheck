## 프로젝트 한 줄 요약

**PawCheck**: 반려동물 동반 시설의 출입 규정(공사 OpenAPI 원문)을 구조화된 체크리스트로
변환하고, 사용자의 반려동물 조건에 맞춰 "입장 가능 여부"를 사전에 예측해 헛걸음을 줄이는
웹 서비스. 2026 관광데이터 활용 공모전 ②-2 웹·앱 구현 부문 지정과제 6번
("모호한 반려동물 출입 조건 및 규정 인지 오류로 현장 입장 거부 및 헛걸음 유발 문제") 대응.

## 반드시 지켜야 할 것

- **한국관광공사 OpenAPI(KorPetTourService2) 필수 활용** — 이 API 없이 더미 데이터로 대체하지 말 것.
  단, 개발 초기 단계(서비스키 미발급/네트워크 미연결 시)에는 `docs/03_데이터명세.md`에 있는
  실측 샘플 응답을 fixture로 사용해 UI/로직을 먼저 완성하고, 이후 실제 키 연동으로 전환할 것.
- **일 1,000건 API 트래픽 제한**을 고려해 반드시 DB 캐싱 계층을 둘 것. 같은 콘텐츠를 매 요청마다
  다시 호출하지 말 것.
- `relaAcdntRiskMtr` 필드는 실측 결과 `acmpyPsblCpam`과 값이 중복/오염되어 있음이 확인됨
  (03_데이터명세.md 참고). **이 필드는 신뢰하지 말고 UI에 노출하지 말 것.**
- LLM 파싱(등급/예외조항 추출)은 `etcAcmpyInfo` 필드에 한정할 것. 나머지 필드는 규칙 기반 파서로
  충분하므로 굳이 LLM을 거치지 말 것 (비용/속도/일관성 이유).
- 완성된 결과물은 "실행 가능한 웹 서비스"여야 함 — 즉 `npm run dev`로 로컬에서 바로 동작해야 하고,
  목업이나 정적 스크린샷 수준이 아니라 실제 데이터 흐름(API → 파싱 → DB → UI)이 동작해야 함.

## 기술 스택 (고정값 — 임의 변경 금지)

- Next.js 14+ (App Router) + TypeScript
- TailwindCSS
- Prisma + SQLite (로컬 파일 DB, 별도 서버 설치 불필요)
- Anthropic API (`@anthropic-ai/sdk`) — `etcAcmpyInfo` 파싱 전용
- 인증/로그인 없음 (닉네임 기반 익명 후기만 지원, MVP 범위 최소화)

## 환경변수

`.env.example`을 `.env`로 복사해서 사용. 필요한 키:

```
TOUR_API_SERVICE_KEY=   # 공공데이터포털에서 발급받은 KorPetTourService2 서비스키 (Decoding 값)
ANTHROPIC_API_KEY=      # etcAcmpyInfo 예외조항 파싱용
```

키가 없는 상태에서도 fixture 데이터로 앱이 돌아가야 하므로, API 클라이언트는
키 미설정 시 `docs/03_데이터명세.md`의 샘플 응답을 반환하는 mock 모드로 자동 전환할 것.

## 작업 방식

- 한 번에 전체를 만들려 하지 말고 `docs/05_개발로드맵.md`의 Phase 순서를 따를 것.
- 각 Phase가 끝나면 해당 Phase의 완료 조건(DoD)을 스스로 점검하고 다음 Phase로 넘어갈 것.
- 애매한 부분은 이 문서와 `docs/` 폴더 전체를 다시 확인한 뒤 가장 합리적인 기본값으로 진행할 것
  (사용자에게 되묻지 말고 문서에 근거해 판단).
