# 04. 화면설계 — PawCheck

## 1. 화면 목록 (라우트)

| 라우트 | 화면명 | 설명 |
|---|---|---|
| `/` | 홈 | 서비스 소개 + 지역/카테고리 검색 진입 + 내 반려동물 프로필 설정 |
| `/facilities` | 시설 목록 | 검색/필터 결과 카드 리스트 |
| `/facilities/[id]` | 시설 상세 | 체크리스트, 예외조항, 맞춤 판정, 후기, 출발 전 체크 |
| `/facilities/compare` | 규정 비교 | 선택한 2~3개 시설 체크리스트 나란히 비교 |
| `/pet-profile` | 내 반려동물 설정 | 견종/크기/맹견여부/목줄·입마개 소지 여부 입력 (localStorage 저장) |

## 2. 사용자 플로우

```
[홈]
  ├─ (최초 방문 시) 내 반려동물 프로필 입력 유도 → localStorage 저장
  ├─ 지역/카테고리 선택 → [시설 목록]
  │                             ├─ 카드 클릭 → [시설 상세]
  │                             │                 ├─ 체크리스트 + 맞춤 판정 확인
  │                             │                 ├─ 출발 전 체크리스트 확인
  │                             │                 ├─ 최근 방문자 후기 확인/작성
  │                             │                 └─ "비교에 추가" → [규정 비교]
  │                             └─ 다중 선택 → [규정 비교]
  └─ [내 반려동물 설정] 언제든 재진입 가능 (헤더 고정 링크)
```

## 3. 디자인 톤

- 컬러: 따뜻하고 안심되는 톤 (예: 민트/코럴 계열 포인트 컬러 + 뉴트럴 그레이 배경).
  경고성 빨강은 "❌ 불가" 표시에만 제한적으로 사용
- 판정 배지는 색상만이 아니라 아이콘+텍스트를 함께 사용 (색맹 접근성 고려):
  - ✅ 가능 (초록)
  - ⚠️ 조건부 (노랑/주황)
  - ❓ 확인필요 (회색)
  - ❌ 불가 (빨강)
- "확정 단정" 대신 "현재까지 확인된 정보 기준"이라는 뉘앙스를 전 화면에 일관 적용

## 4. 화면별 상세

### 4-1. 홈 (`/`)

- 히어로 섹션: 서비스 한 줄 소개 + CTA("내 반려동물로 시작하기")
- 반려동물 프로필 미설정 시 배너로 설정 유도 (스킵 가능, 스킵 시 판정은 "확인필요"로만 표시)
- 지역 선택(시/도 → 시/군/구 드롭다운, 법정동 코드 매핑은 03-data-spec.md 참고 API 별도 호출)
- 카테고리 선택: 관광지 / 레포츠 (MVP), 숙박·음식점은 "곧 추가됩니다" 배지로 표시만

### 4-2. 시설 목록 (`/facilities`)

- 필터 바: 지역, 카테고리, "실내 가능만 보기", "맹견 제한 없음만 보기" 토글
- 카드 구성: 대표이미지, 시설명, 주소 요약, 배지 3개(실내/실외/견종제한), 신뢰도 지수 뱃지
- 정렬: 최신 수정순 / 신뢰도순
- 빈 상태(필터 결과 0건): "조건에 맞는 시설이 아직 없어요 — 커뮤니티에 등록을 제안해보세요"
  (MVP에서는 텍스트만, 실제 제안 폼은 후속 버전으로 명시)

### 4-3. 시설 상세 (`/facilities/[id]`)

상단부터 순서대로:
1. 기본 정보: 이름, 주소(지도는 좌표만 표시, 실제 지도 임베드는 선택 구현)
2. **내 반려동물 기준 판정 카드** (가장 눈에 띄게):
   - 배지(가능/조건부/확인필요/불가) + 판정 사유 1~2문장
   - 예: "⚠️ 조건부 가능 — 맹견은 입마개 착용 시 입장 가능합니다. 입마개를 챙겨오세요."
   - 반려동물 프로필 미설정 시: "내 반려동물 정보를 입력하면 맞춤 판정을 볼 수 있어요" CTA
3. **표준 체크리스트** (표 형태, 02-design.md §4 구조 그대로 렌더링)
4. **예외조항** (LLM 파싱 결과, `exceptions` 배열): "⚠️ 실내 미술관은 동반 불가" 같은 카드형 목록
5. **출발 전 준비물**: `relaFrnshPrdlst`(시설 비치) vs `relaRntlPrdlst`(대여 가능) 구분해서
   "챙겨올 필요 없어요(시설 제공)" / "여기서 빌릴 수 있어요" / "직접 챙기세요" 3분류로 안내
6. **최근 방문자 후기**: 리스트(닉네임, 방문일, 일치여부 배지, 코멘트) + "방문 확인하기" 버튼
   (모달: 방문일 선택 + 규정이 그대로였는지 O/X + 코멘트 입력)
7. 신뢰도 지수 표시 (02-design.md §7 로직)

### 4-4. 규정 비교 (`/facilities/compare?ids=A,B,C`)

- 선택된 시설들을 열(column)로, 체크리스트 항목을 행(row)으로 배치한 비교표
- 최대 3개까지 비교 가능, 4개 이상 선택 시 안내 토스트

### 4-5. 내 반려동물 설정 (`/pet-profile`)

- 입력 항목: 견종(자유입력 or "모르겠어요"), 맹견 해당 여부(체크박스 + 안내 툴팁),
  목줄 소지 여부, 입마개 소지 여부
- 저장 즉시 localStorage에 반영, 별도 로그인/서버 저장 없음
- "이 정보는 이 브라우저에만 저장되며 서버로 전송되지 않습니다" 안내 문구 (신뢰 확보)

## 5. 공통 컴포넌트

- `<PolicyBadge status="가능|조건부|확인필요|불가" />`
- `<PetPolicyChecklist policy={...} />`
- `<TrustScoreBadge reviews={...} />`
- `<ReviewForm facilityId={...} />`
- `<PetProfileBanner />` (미설정 시 전역 유도 배너)

## 6. 접근성/신뢰성 관련 필수 준수사항

- 모든 자동판정 옆에는 반드시 "원문 보기" 토글을 두어, 공사 API 원문 텍스트를 그대로
  확인할 수 있게 한다 (오파싱 시 사용자가 직접 검증 가능하도록)
- 판정 근거가 애매한 경우 절대 "가능"으로 단정하지 않고 "확인필요"로 표시 (안전 우선)
